错误解决

1.项目持续在云服务器执行，使用screen依赖

[Linux系统SSH客户端断开后保持进程继续运行配置方法 - 阿里云 (aliyun.com)

Linux系统SSH客户端断开后保持进程继续运行配置方法 - 阿里云 (aliyun.com)

Linux screen命令 | 菜鸟教程 (runoob.com)

2.Caused by: java.lang.IllegalArgumentException: Property ‘sqlSessionFactory’ or ‘sqlSessionTemplate’ are required

mybaits包倒错了，mysql-connetr-java依赖导入，原文链接找不到了

3.依赖自带拦截器去除

自带拦截页面和登录页面，非常**，我都不知道哪来的404，403

4.@ComponentScan(basePackages = { “cn.eli.vue” })扫描接口，启动器同目录或扫描接口目录

否则可能找不到mapper层的接口

5.content-Type ： application/json

会报415，说明出现了数据包的格式或者表头错误，数据包一定要是 content-Type ： application/json的header

6.


${basedir}/src/main/java

**/*.xml



这样子构建了之后，mapper.xml中的sql才能执行

7.@Reqestbody接受不到postman传数据包
entity类中，不能用static类型***

8.jar包 无主属性清单
跳过测试 使用maven打包，并将skip置为true

9.传输到服务器发布
ftp root权限，设置777权限文件夹，userlist_deny=NO,

ftp 503 https://blog.csdn.net/cmustard/article/details/53696456

10.Could not autowire. No beans of ‘xxxx’ type found
@Autowired注释问题，不影响编译

11.Mapper method ‘cn.eli.vue.mapper.UserMapper.Register’ has an unsupported return type: class cn.eli.vue.entity.User

接口的类型要和sql语句的返回值类型一直，insert sql 语句的返回值？
mysql中 ：insert sql 语句的返回值为 {1,0} 因此该接口类型应当为整形
